UI설계
화면정의서
왼쪽에 홈페이지의 화면이 캡쳐해 넣거나 그린다.
오른쪽에 디스크립션(description)을 넣는다.
디스크립션에는 홈페이지의 동작의 설명을 넣는다.

화면정의서를 읽을 수 있고, 작업을 할 수 있어야 함

UI/UX 콘셉트 기획
내가 만드려는 작업물의 동기,기대효과를 기획서같이 만드는것

UI(User Interface) : 사용자가 직접 보는 화면 디자인
UX(User eXperience) : 그 제품을 사용한 사용자의 느낌

콘셉트 기획을 하는 이유
- 사용자의 불편함을 해결할 수 있도록 서비스를 설계하는 과정

ex) 쇼핑몰을 만드려고 한다.

컨셉을 기획하는법

1. 사람들한테 조사를 받는다.(설문조사, 구글폼 등등)
- 연령대,성별,기/미혼
- 어떤 쇼핑몰을 주로 사용하냐
- 어떤 기기를 주로 사용하냐
- 어떤 제품을 주로 구입하는가
- 불편했던점

2. 페르소나(가상의 인물)
- 리서치 결과를 바탕으로 가상의 인물을 생성
- 인물의 이름, 연령, 직업, 관심사, 목표를 정한다.

3. 사용자 시나리오
- 가상의 인물이 내가 만든 서비스를 이용하는 형태를 구술
- 사용자 목표 정의(사용자가 원하는게 무엇인지)
- 사용자의 행동 흐름을 작성(사용자가 로그인을 하고 상품을 검색하여 장바구니에 담고 결제를 하기까지의 과정을 나열)
- 각 단계에서의 사용 경험(UX) 개선

UI 아키텍처 설계

플로우 차트(flow chart) : 작업의 흐름을 도형으로 나타난것

터미널로 시작 흐름선으로 흐름을 잇는다.
입출력이 필요하면 입력/출력 도형을 사용한다.
입출력후 판단도형을 이용하여 검증한다.
ㄴ 참일 때의 흐름, 거짓일때의 흐름 두가지가 존재
참일때의 흐름이라면 처리도형을 통해 처리한다.
터미널로 마무리한다.
